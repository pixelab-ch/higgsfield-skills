  higgsfield:presets_show        / mcp__higgsfield__presets_show

  higgsfield:select_workspace    / mcp__higgsfield__select_workspace

Naming rule (D-03): colon form in prose/SKILL.md body; double-underscore form in
  `allowed-tools` frontmatter. Same tool, different syntactic context.

Tool purposes / returns (STACK.md section 5):
  generate_image / generate_video -> submit job, return job_id; params are model-dependent,
    always call models_explore first. input_files carries uploaded media IDs (roles: image,
    start_image, end_image, video, audio).
  models_explore -> live per-model schema (parameters, aspect_ratios, durations/duration_range,
    media roles). Source of truth; call before generating.
  job_status -> one-shot status check (pending/running/complete/failed) + result URL.
  job_display -> render job result with output URL (poll-friendly display).
  media_upload -> returns pending media ID. media_confirm -> returns usable confirmed ID.
    Atomic pair, used before passing local media to generate_*.
  presets_show -> list presets (for higgsfield_preset model).
  show_plans_and_credits -> plan upgrades + credit top-ups. balance -> current credit balance.
  select_workspace -> switch private/shared workspace (situational).

Canonical flow (ARCHITECTURE.md Pattern 2; D-04, D-05):
  1 Resolve model (read skill model-specs.md, match routing table, clarify if ambiguous)
  2 Build prompt (apply skill prompt-craft, validate params vs model constraints)
  2b [conditional] If user supplies local media OR model needs start_image/audio:
     media_upload -> media_confirm -> use returned ID in input_files (atomic pair)
  3 Confirm [REQUIRED]: show model + rationale, prompt, key params, estimated credit cost
     (balance / show_plans_and_credits); wait for explicit user YES. No generate_* before this.
  4 Generate: higgsfield:generate_image or higgsfield:generate_video -> capture job_id
  5 Poll: higgsfield:job_status with job_id, backoff 30-60s; NEVER re-call generate_* to poll
  6 Display: higgsfield:job_display when complete; report job_id + asset URL
